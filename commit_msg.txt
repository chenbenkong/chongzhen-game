feat(ui): 古风确认弹窗 ConfirmDialog 替换 window.confirm

新建组件：
- ConfirmDialog.tsx + .css：暗金古风主题确认弹窗，Portal 到 body
- useConfirm.tsx hook：返回 { confirm, dialog }
  用 Promise 替代 window.confirm 的同步阻塞调用
  confirm({ title, message, detail?, warning?, variant?, confirmText?, cancelText? })
  -> Promise<boolean>

弹窗设计：
- 标题（暗金色 + Ma Shan Zheng 字体）
- 详情区（如果有事件关联，显示 "📜 关联事件：xxx"）
- 警告区（红色 "⚠ 焚毁后无法复原"）
- 两按钮：保留（暗金） / 焚毁（暗红 danger 样式）
- 渐入动画（0.2s cubic-bezier(0.16, 1, 0.3, 1)）
- Esc 取消 / Enter 确认（无 input focus 时）
- 锁 body 滚动
- 默认聚焦"焚毁"按钮
- 圆角 + 阴影 + 边框符合整体暗金古风
- z-index 11000（最高），脱离父 stacking context

替换了三处 window.confirm：
1. ImageGenerator handleDelete：画册单图删除
   文案改为古风："焚毁此画" / "焚毁" / "保留"
2. ImageGenerator handleClearAll：清空全部
   文案："焚尽卷轴阁" / "全部焚毁"
3. EventImages handleDelete：事件插图删除
   自动带 detail = "📜 关联事件：xxx"

构建时修了一个小坑：useConfirm.ts 改为 .tsx（内含 JSX）
